# Parametric study example
This example shows how to perform frequency scans with different system parameters. The goal is to capture the potential operating-point dependency of power converters' dynamics. Then, it applies interpolation to estimate the admittance at different power flow conditions. Lastly, 3D admittance plots are generated to visualize the differences between scenarions. The code can be easily adapted to evaluate the sensitivity with respect to other quantities such as control parameters. The project contains the same model used in the [2L-VSC](../2L_VSC/README.md) example.

## Setting up the PSCAD model
It is assumed that the [pre-requistes](../README.md) are installed. Similarly to the [2L VSC example](../2L_VSC), when opening the [Parametric_study.pswx](Parametric_study.pswx) PSCAD workspace in this folder, the Z-tool library will appear grayed-out as it points to the computer where it was last saved. Therefore, in the PSCAD project simply **unload** the grayed-out library by right-cliking on it and selecting _Unload_, then **add the library** file _Z_tool.pslx_ within the Z-tool installation path in your PC (_Scan_ folder at the directory retrieved by cmd `py -m pip show ztoolacdc`), **move it up** before your project files and **save** the changes.

The provided model already contains an AC scan block placed at the Point of Common Coupling (PCC) of the inverter model under analysis as seen below. The name of the scan block is PCC and will be used to retrieve the scan results. The inverter is at side 1 of the block, while an RLC equivalent is at side 2. In addition, two groups of parameters are shown at the top:

1. **IBR parameters**: used to set the converter active (_p_ref_) and reactive (_q_ref_) power references. The power measurement filter time constant can also be changed with the _pq_tau_meas_ value (not used in this example).

1. **Grid-side parameters**: the voltage magnitude is defined in per unit (_V_pu_) and converted to kV (V_SI). This value defines the voltage source amplitude. Furthermore, this value is introduced in the _Peak voltage L-G [kV]_ parameter, and the option _Vac-controlling node?_ is set to _Yes_ so as to force the steady-state voltage at the scan point to the provided value.

The operating point of the converter can be varied as desired by changing these values. In the next step, the `frequency_sweep` function will automatically set these parameters so as to scan the converter at different terminal conditions.

## Basic simulation and scan options
Once the PSCAD model is setup, the scan parameters need to be provided in the corresponding python script [parametric_sweep_scan.py](parametric_sweep_scan.py). Some arguments are similar to those in the [2L VSC example](../2L_VSC), such as `num_parallel_sim` and `multi_freq_scan`, but additional functionalities are now used to perform the scan after changing the desired parameters. In particular, the desired operating points are defined in terms of  voltage magnitude and powers. Then, all combinations of said values are generated by using Numpy's `meshgrid` function. Note that the total number of cases grows very fast with increased number of parameters and their values. The rest of the script loops over each scenario, performs the scan after setting the parameters in PSCAD and appends each flattened admittance (by means of the `.ravel(order='C')` method) together with the operating point to a single results file (`results_file`).

The most relevant arguments compared to other examples are explained below:
1. **The _component_parameters_** argument: this argument expects a list containing pairs of ["Name", _value_] where the "Name" refers to a constant name in the PSCAD project canvas, and _value_ is the desired float value. In this case, the voltage magnitude and powers are defined. For example, `component_parameters=[["V_pu", 1.0], ["p_ref", 0.5], ["q_ref", 0.1]]` sets the voltage magnitude to 1.0 in per unit, and the active and reactive power to 0.5 and 0.1 in per unit, respectively.
2. **PSCAD handling** options: used to improve the application control to minimize loading time and maintain the user-defined license type during all scans
    * `release_certificates=False` forces to retain the license even after the function is executed
    * `delete_PSCAD_output_files=False` does not delete the files generated during the simulations to avoid unnecessary model re-compilation time
    * `launch_and_load_PSCAD=True` for the initial case it opens PSCAD and loads the specified project, while for all the other cases `launch_and_load_PSCAD=False`, which results in the usage of the previously opened instance of PSCAD. To avoid closing PSCAD after each scan is finished, the argument `close_PSCAD` is set to `True` only when the last scan is about to be performed,while `close_PSCAD=False` otherwise. Therefore, opening and closing the application needlessly is avoided, and performance is improved.
3. **Admittance plotting** (`make_plot` argument): this boolean argument is set to `False` to avoid the plotting of a large number of results. Instead, the results are more efficiently saved as tab-separated text files.
3. **Power flow display** (`show_powerflow` argument): this boolean argument is set to `False` to avoid the printing of the steady-state power flow after each scan is performed.

For more information on the function arguments, open a python shell, import the function e.g. `from ztoolacdc.frequency_sweep import frequency_sweep`, and type `help(frequency_sweep)`.

## Parametric frequency scan
The script [parametric_sweep_scan.py](parametric_sweep_scan.py) loops over all the generated cases with different operating points, runs the frequency scan using previously described arguments and appends the admittance into a single file. These results can be used to study how the operating point changes influence the small-signal stability in a given system.
When the scan data for a specific operating point is not known, linear interpolation can be applied based on the dynamics at other operating conditions. The script uses scipy's `LinearNDInterpolator` linear interpolation function for this purpose. The figure below shows the cloud of scanned operating scenarios in blue, and a new operating point in red for which the admittance is not known and interpolation is applied.

![Parametric_study_scenarios](../../Doc/Parametric_study_1.png)

Lastly, the actual admittance at the new operating point is scanned and compared with the interpolated results to verify the adequacy of the interpolation. As seen below, the minor differences between the interpolated (dots) and actual (lines) admittance emphasize the applicabilability of interpolation.
